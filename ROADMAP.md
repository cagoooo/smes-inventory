## 📋 2026 全校盤點 SOP（依目前系統能力）

> **致資訊組老師**：以下流程是用 v7.3.1 能實現的最省力全校盤點方式。

### 階段 1：事前準備（30 分鐘）
1. **印 QR 標籤**：管理後台 → 🏷 QR 標籤 → 篩選要印的財產 → 產生 PDF → 用 A4 貼紙印
2. **每班貼上 QR**：把對應財產的 QR 貼到主機邊、螢幕底座、觸屏側邊（統一位置）
3. **熟悉拍照 SOP**：
   - 主機：拍機身商標 + 機型銘牌（側面）
   - 觸屏：拍右下角品牌/型號貼紙
   - 財產標籤：正面清楚對焦

### 階段 2：每班現場拍照（每班 2-3 分鐘）
```
進教室 → 選教室（或掃 QR 自動帶入） 
  ↓
開「連續拍照模式」開關
  ↓
【第一台主機】
  ① 拍主機正面（商標+型號）→ AI 辨識
  ② 檢查 diff panel 有沒有跳出「教室位置不符」或「廠牌不符」
  ③ 勾選要更新的欄位 → 儲存
  ↓
【自動跳第二台】
  同樣流程...
  ↓
【拍完全班】→ 換教室
```

### 階段 3：異常處理
| 症狀 | 原因 | 解法 |
|------|------|------|
| 🚛 藍色 banner「登記在其他教室」| 清冊錯 / 搬家 / 未更新 | 勾選 → 儲存 → 自動移轉 |
| 🔍 紅色「找不到對應」+ 有財產號 | 新購機或清冊沒這筆 | 按「➕ 新增為新財產」 |
| ⚠️ AI 辨識信心度低 | 光線不好 / 角度偏 | 重新拍 (Live 相機模式較穩) |
| 📴 網路不穩 badge 變紅 | 訊號差 | 不用擔心，會進離線佇列自動重傳 |

### 階段 4：完成後確認
1. 管理後台 → 🏫 教室 → 查看每間「📷」的數字是否跟實際台數一致
2. 管理後台 → 📦 財產 → 搜尋「更新時間今天」找不到的 → 可能是沒拍到
3. 產出本月月報表 PDF（📈 儀表板頂部按鈕）給主任

### 階段 5：每學期例行
- 每月 1 日產月報表
- 每學期初與財管系統對帳一次（目前手動，v9 後可自動）
- 每學年結束做一次全校普查

### 🔬 v7.3.3 — AI 辨識流程審計 + 精準度升級 (2026-04-18)

> 檢查 AI 辨識 → 欄位對應 → 清冊比對的完整流程，找到 7 個潛在問題，修正最關鍵的 4 個。

**發現的問題**
- 🔴 `searchInventory` 原本是 fuzzy 全欄比對，財產號 `0001` 會誤配到 model 含 `0001` 的紀錄
- 🔴 `confidence` 欄位完全沒呈現，AI 信心度 30% vs 95% 看起來一樣
- 🔴 AI 回傳 `is_old_device` 判斷從沒被使用
- 🔴 AI 認到財產號但查不到時，只顯示「新增」按鈕，沒考慮是「辨識錯誤」
- 🟡 所有照片類型用同一個 prompt（待改進）
- 🟡 AI 辨識結果沒地方預覽（待改進）

**修正內容**
- ✅ **多階段精確比對**：`findInventoryByExactPN` → `findInventoryByFuzzyPN` → 全欄 fuzzy
  - 先 exact match on property_number
  - 失敗才用 OCR 混淆字元（1↔7 / 0↔O / B↔8 / 5↔S / 2↔Z）產生變體再查
  - 最後才 fallback 到全欄模糊
- ✅ **信心度條 UI**：表單頂部彩色 bar
  - 🟢 ≥85% 高信心 → 「可以直接儲存」
  - 🟡 65-85% 中信心 → 「建議檢查欄位」
  - 🔴 <65% 低信心 → 「強烈建議重拍」
  - 列出 AI 沒讀到的欄位（例：「未讀到：廠牌 / 型號」）
- ✅ **`is_old_device` 整合到 notes**
  - AI 判斷老舊 → 自動附加「⚠️ AI 判斷外觀已老舊（機殼泛黃/磨損/髒污），建議納入汰換評估」
- ✅ **財產號查無時的雙選擇 UI**
  - 不再只給「➕ 新增」—改為列 3 個可能原因
  - 兩顆按鈕：「✏️ 我要修正財產編號」vs「➕ 確認是新財產」
  - 「修正」按鈕會 auto-focus + select 財產號輸入框
- ✅ **OCR 模糊比對提示**
  - 若是靠 OCR 混淆字元才找到 → banner 顯示「🔤 用 OCR 常見混淆字元找到相近 N 筆，請仔細確認」
  - 若是靠全欄 fuzzy → 灰色 hint 「僅供參考」

### 🔄 v7.2.9 — 拍照即時同步更新財產清冊 (2026-04-18)

> 問題：使用者發現「這幾年很多班級電腦故障壞掉直接換主機但沒更新清冊」，想靠這次盤點全面更新。但原本流程只寫入 `photo_records`，**完全不更新 `inventory_items`**，所以就算 AI 認出新型號，清冊仍是舊的。

**DB 層**
- ✅ `inventory_items` 新增 `serial_number` 欄位（AI 讀得到的 S/N 也能存）
- ✅ `inventory_items.updated_at` 新增自動更新 trigger
- ✅ `supabase-client.js` 新增 `updateInventoryItem(id, patch)` + `insertInventoryItem(item)` API

**流程改造**
- ✅ **自動偵測差異**：選到 match 建議後，比對 AI 辨識 vs 清冊現值
  - 比對欄位：`brand / model / acquired_year / serial_number / classroom_code`
  - 若欄位不同 → 顯示彩色 diff 面板（紅刪除線舊值 → 綠色新值）
  - 預設全部勾選（反正 AI 辨識 = 現場實況）
- ✅ **一鍵同步**：按「儲存」時：
  1. 正常 insert `photo_records`（歷史永久保留）
  2. 若有勾選的差異欄位 → PATCH `inventory_items` 對應記錄
  3. Toast 顯示：「已儲存 + 同步更新財產清冊 N 個欄位」
- ✅ **找不到對應時新增**：AI 辨識到財產號但清冊沒這筆 → 顯示「➕ 新增為新財產」按鈕，一鍵建立
- ✅ **即時 diff 重算**：使用者手動編輯表單 → 300ms debounce 重新計算差異

**使用情境**
| 場景 | 流程 | 清冊結果 |
|------|------|---------|
| C308 舊主機已換新（Lenovo → ASUS） | 拍照 → AI 認 ASUS → 自動選舊 match → diff 顯示 3 個差異 → 勾選儲存 | ✅ 清冊從 Lenovo 改為 ASUS |
| 新到的主機沒在清冊 | 拍照 → AI 認財產號 → 找不到 → 按「新增為新財產」 | ✅ 清冊新增一筆 |

### #1 📜 異動歷史記錄表 `inventory_audit_log`（**最高優先**，2h）
**為什麼急**：您現在可以拍照更新清冊（v7.2.9）+ 自動搬移教室（v7.3.0），但**改了就改了**，沒有回溯機制。半年後您若要查「C308 的主機是什麼時候從 Lenovo 換成 ASUS 的？」完全無從得知。

**做法**：
```sql
CREATE TABLE inventory_audit_log (
  id SERIAL PRIMARY KEY,
  inventory_id BIGINT REFERENCES inventory_items,
  changed_by UUID REFERENCES auth.users,
  changed_at TIMESTAMPTZ DEFAULT NOW(),
  field_changed TEXT,     -- 'brand' / 'model' / 'classroom_code'
  old_value TEXT,
  new_value TEXT,
  source TEXT             -- 'photo_recognition' / 'manual' / 'excel_import'
);
```
app.js 在 updateInventoryItem 前先 INSERT log → DB 層或 app 層都可。

### #2 📊 本次盤點異動總覽報表（1.5h）
**為什麼急**：您跑完一輪後，想知道「這次我改了什麼」才能跟主任交代。現在只能一筆筆去資料庫查。

**做法**：管理後台加「📋 盤點報告」分頁，查詢 updated_at ≥ 本月 + photo_records 聯表，產出：
- 本月新增財產 N 台（列表）
- 本月更新資料 M 台（哪些欄位被改）
- 本月搬動設備 K 台（從哪到哪）
- 本月發現異常（信心度低 / 清冊錯 / 設備遺失疑慮）

### #3 🎯 盤點進度地圖（1h）
**為什麼急**：您現在平面圖顯示「有拍 vs 沒拍」是歷史累積的，分不清哪些是「本次盤點拍過」。一邊跑會不知道今天進度到哪。

**做法**：平面圖加開關「顯示本次盤點進度」：
- 🟢 本月內有拍過
- 🟡 本月沒拍但以前拍過
- 🔴 從未拍過
- 白 無財產

做完這 3 件 → 下一波「全校盤點」您會感到系統真的幫到您，而不是「拍完後不知道成效」。
